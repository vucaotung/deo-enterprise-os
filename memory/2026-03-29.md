## Technical Learnings

### Database Schema Reality
- `deo.companies` table has columns: id, name, code, created_at, updated_at (NO description, NO is_active)
- `deo.business_lines` table has: id, name, code, description, is_active, created_at, updated_at
- `deo.company_business_lines` junction table: company_id, business_line_id, is_primary, created_at
- `deo.agent_assignments`: id, agent_name, company_id, scope, is_active, created_at, updated_at
- `deo.staff_assignments`: id, user_id, company_id, role, is_active, created_at, updated_at

### WSL/Windows File Sync Issues
- Writing files to Windows path (C:\...) does NOT immediately sync to WSL filesystem
- Solution: Use `/mnt/c/...` path when running node scripts from WSL
- NODE_PATH trick needed when running scripts outside project: `NODE_PATH=./node_modules node /mnt/c/...`

### Seed Script Approach
- Cannot use psql directly from Windows (not in PATH)
- Cannot use psql from WSL (not installed in Ubuntu WSL)
- Solution: Node.js script with pg module, run from API project context where pg is installed
- Must match exact schema columns (no description/is_active for companies)
