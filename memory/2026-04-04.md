## Deploy deo-enterprise-os lên VPS + Fix dashboard login

### Context
Sếp yêu cầu deploy hệ thống deo-enterprise-os lên VPS Contabo và setup 5 Telegram bots với webhook integration.

### VPS Info
- IP: `173.249.51.69`
- Domain: `enterpriseos.bond`
- Dashboard: `https://dash.enterpriseos.bond`
- API: `https://api.enterpriseos.bond`

### Công việc đã hoàn thành

#### 1. Telegram Webhook Integration
- Tạo route `/api/telegram/webhook/:botName` trong `apps/api/src/routes/telegram.ts`
- Setup webhook thành công cho 5 bots:
  - `main` / `@condeobot` → `https://api.enterpriseos.bond/api/telegram/webhook/main`
  - `admin` / `@agent_admin_deo_bot` → `.../webhook/admin`
  - `phapche` / `@agent_phapche_deo_bot` → `.../webhook/phapche`
  - `ketoan` / `@agent_ketoan_deo_bot` → `.../webhook/ketoan`
  - `dieuphoi` / `@agent_dieuphoi_deo_bot` → `.../webhook/dieuphoi`
- Bot tokens lưu trong `.env` server-side với prefix `TELEGRAM_BOT_*`
- Commands đã implement: `/start`, `/task`, `/expense`, `/client`
- Bot echo lại message thường và ghi vào database thật

#### 2. Dashboard Login Fix
**Vấn đề:** Dashboard không login được vì frontend gọi sai API URL.

**Root cause:** 
- Frontend build với `VITE_API_URL` từ `.env.production`
- Dockerfile **không copy** file `.env.production` vào build context
- Vite fallback về `http://localhost:3001/api` thay vì `https://api.enterpriseos.bond/api`

**Giải pháp:**
1. Sửa `apps/web/Dockerfile` → thêm dòng `COPY apps/web/.env.production ./`
2. Rebuild web container với env var đúng
3. Restart container

**Thông tin đăng nhập dashboard:**
- Username: `admin`
- Password: `admin123`
- Hardcoded trong `apps/api/src/middleware/auth.ts`

#### 3. Files đã sửa
- `/opt/deo-enterprise-os/apps/web/Dockerfile` → thêm COPY .env.production
- `/opt/deo-enterprise-os/apps/web/.env.production` → `VITE_API_URL=https://api.enterpriseos.bond/api`
- `/opt/deo-enterprise-os/apps/api/src/routes/telegram.ts` → webhook handler + DB write
- `/opt/deo-enterprise-os/.env` → 5 bot tokens

### Lessons Learned
- **Vite env vars:** Phải copy `.env.production` vào Dockerfile, không thì build-time env vars không có
- **Docker multi-stage build:** Env file phải có ở builder stage, không phải runtime stage
- **Debug frontend API calls:** Check DevTools Network tab để thấy frontend đang gọi URL nào
